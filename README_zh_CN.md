# 日期前缀刷新

一个很小的思源插件：根据文档内部的 `updated` 字段，把文档标题刷新成 `YYMMDD-标题`。

例如：

```text
各模型聚合API → 260526-各模型聚合API
250901-各模型聚合API → 260526-各模型聚合API
```

## 为什么做这个插件

很多人会在文档标题前手动写更新时间，比如 `260526-各模型聚合API`。但这个日期本质上是一个“派生字段”：文档真实更新时间已经存在于思源内部元数据里，再手动维护一次，迟早会不一致。

这个插件就是把这件事自动化。

## 功能

插件启用后提供两个入口：

- 顶栏按钮：刷新文档日期前缀
- 命令面板：搜索“刷新文档日期前缀”

点击后会弹出范围选择：

- 最近 7 天
- 最近 30 天
- 当前笔记本
- 全部笔记本

插件会把符合条件的文档标题改成：

```text
YYMMDD-原标题
```

如果原来已经有六位日期前缀，会替换旧前缀，不会叠加。

## 安全规则

插件永远跳过目录型文档。

思源的目录本身也是文档。只要一个文档下面还有子文档，本插件就认为它是“目录型文档”，不会改名，避免破坏你的文档树结构。

插件还会跳过：

- 日记标题，例如 `2026-05-28`
- `未命名`
- `未命名文档`
- 没有合法 `updated` 字段的文档
- 前缀已经正确的文档

## 数据来源

插件使用思源数据库里的 `blocks.updated` 字段，格式通常是：

```text
YYYYMMDDHHmmss
```

标题前缀取其中的 `YYMMDD`。

## 安装

### 从插件市场安装

如果插件已经上架思源插件市场，直接在思源集市中搜索：

```text
日期前缀刷新
```

### 手动安装

下载 Release 里的 `package.zip`，解压到：

```text
工作空间/data/plugins/siyuandateprefix
```

然后重启思源，在插件管理中启用。

## 开发与打包

本插件是纯 JavaScript 插件，无需构建。

打包：

```bash
zip -r package.zip plugin.json index.js index.css README.md README_zh_CN.md LICENSE icon.png preview.png
```
